## CORE FEATURE 1: DOCUMENT INGESTION + OCR PIPELINE

Support these upload types: PDF, DOCX, TXT, PNG, JPG, TIFF, XLSX, CSV, audio (mp3/wav via Whisper), URLs (web scrape).

Pipeline per document:
1. Upload file → store in AWS S3, record in Postgres (documents table)
2. Detect document type
   - If image or scanned PDF → route to AWS Textract
   - AWS Textract must extract: plain text, tables (as markdown), form key-value pairs, handwriting
   - For native text PDF/DOCX → parse directly
   - For XLSX/CSV → convert to markdown table
   - For audio → transcribe via OpenAI Whisper
   - For URLs → use Cheerio to scrape and clean HTML
3. Split into chunks using LangChain RecursiveCharacterTextSplitter (chunk_size: 1000, overlap: 200)
4. Generate embeddings via OpenAI text-embedding-3-large
5. Upsert all vectors to Pinecone with metadata: {doc_id, chunk_index, page_number, source_type, char_start, char_end}
6. Store chunk metadata in Postgres (chunks table)
7. Generate and store a document summary (3-sentence, bullets, abstract formats) using LangChain summarize chain
8. Extract and store: document title, author, date, key entities (people, orgs, dates, places) via LangChain entity extraction chain
9. Mark document as "ready" and notify the user

Show real-time progress with a status stepper (Framer Motion): Upload → OCR → Chunking → Embedding → Indexing → Ready

## CORE FEATURE 2: LANGCHAIN RAG AGENT

Build a LangChain AgentExecutor with the following tools. The agent decides which tools to call per query.

### Tool: document_retriever
Uses LangChain VectorStoreRetriever with Pinecone. Retrieve top-8 chunks by cosine similarity. Apply MMR (Maximal Marginal Relevance) for diversity. Filter by document scope (current doc or all docs).

Returns: array of {chunk_text, page_number, doc_id, doc_title, similarity_score, char_range}

### Tool: cross_document_retriever
Same as above but always queries across all documents in the session. Used for multi-doc queries.

### Tool: summarizer
Takes a document_id or set of chunk_ids. Returns summary in requested format: "paragraph" | "bullets" | "abstract" | "executive". Uses LangChain MapReduceDocumentsChain for long docs.

### Tool: multi_doc_comparator
Takes two or more document_ids. Compares them on: key arguments, tone, date/recency, contradictions, shared entities. Returns structured diff.

### Tool: citation_tracer
For any claim in the final answer, trace it back to exact chunk, page number, and document. Format: [Source: DocumentName, Page 4, Paragraph 2].

### Tool: contradiction_finder
Takes chunk_ids from multiple docs. Detects logical contradictions or conflicting data points. Returns list of conflict pairs with explanation.

### Tool: timeline_extractor
Extracts all dated events from chunks, sorts them chronologically, returns structured timeline.

### Tool: entity_extractor
Uses LangChain entity chain to extract people, organizations, locations, dates, monetary values, legislation references. Returns entity list with doc/page provenance.

### Tool: translator
Translates retrieved chunks or the final answer to any language. Uses LangChain LLMChain with translation prompt.

### Tool: quiz_generator
Takes document_id or topic. Generates 5-10 multiple-choice questions with correct answer + distractor explanations. Returns JSON quiz object.

### Tool: hypothetical_qa
Implements HyDE (Hypothetical Document Embeddings). For hypothetical or counterfactual questions, generates a hypothetical document first, embeds it, retrieves real chunks by similarity to the hypothetical, then answers.

### Tool: confidence_scorer
Takes the final answer + retrieved chunks. Scores confidence 0-100% based on: semantic similarity of answer to sources, coverage of retrieved chunks, presence of conflicting chunks. Returns confidence score + explanation breakdown.

### Tool: flashcard_generator
Takes topic or chunk_ids. Returns array of {front, back} flashcards in Anki-compatible format.

Each agent response must include:
- The answer text
- confidence_score (0-100)
- citations (array of {doc_title, page, quote, chunk_id})
- retrieved_chunk_ids (for analytics)
- tokens_used {prompt, completion, total}
- response_time_ms
- tools_called (array of tool names)
- retrieval_scores (array of similarity scores)

25. **Queries per document** (horizontal bar chart): which documents get asked about most
